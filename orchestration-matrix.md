# Orchestration Matrix for BuildToFlip Patterns

The BuildToFlip methodology now formalizes how patterns interact during runtime. This matrix documents interoperability guarantees and the mandatory handoffs that orchestrate collaborative agent behaviors.

## Interoperability Matrix

| Source Pattern            | Target Pattern             | Handoff Contract                                    | Required Telemetry                    |
|---------------------------|----------------------------|-----------------------------------------------------|----------------------------------------|
| Prompt Chaining           | Routing                    | Structured plan with confidence scores              | Step outcomes, routing decision trace |
| Routing                   | Tool Use                   | Intent payload enriched with guardrail assessment   | Tool invocation logs, safety verdicts |
| Tool Use                  | Memory Management          | Structured state delta with provenance metadata     | Memory commit logs, rollback signals  |
| Memory Management         | Reasoning Techniques       | Context snapshot with retrieval evidence            | Retrieval latencies, recall accuracy  |
| Multi-Agent Collaboration | Parallelization            | Role assignments with synchronization checkpoints   | Agent coordination spans              |
| Parallelization           | Guardrails                 | Aggregated result set with anomaly annotations      | Aggregation metrics, anomaly reports  |

## Handoff Protocols

1. **Initiation** – The source pattern issues a `handoff.start` event with trace metadata generated by the observability layer.
2. **Validation** – Guardrails validate payload integrity before the target pattern accepts the handoff.
3. **Acknowledgement** – The target pattern acknowledges via `handoff.ack` and assumes control of the task segment.
4. **Completion** – Results are posted back through `handoff.complete`, enabling ledgering and potential escalation to fallback tiers.

## Interface Contracts

- **Payload Schema** – Every handoff payload must specify `trace_id`, `pattern_origin`, `pattern_target`, `state_delta`, and `confidence` fields.
- **Error Propagation** – Errors propagate through structured `handoff.error` messages so fallback strategies can react deterministically.
- **Telemetry Hooks** – Each pattern registers observability hooks to ensure span-level introspection for every handoff transaction.

This matrix provides the connective tissue between BuildToFlip pattern families, delivering the orchestration discipline required for pragmatic agent deployments.
